# 全网伪科普内容监测预警系统

> 科普中国 · 可行性方案（草案）

---

## 一、问题定义

### 什么是伪科普

不是"科学谣言"。谣言是单条信息的真假判断，伪科普是**系统性的、以科学为包装的有组织传播行为**。

伪科普的核心特征：
- **借科学之名**：使用真实科学概念、术语、论文引用，但结论歪曲或断章取义
- **有目的性**：不是无知导致的错误传播，背后有商业利益、政治动机或组织控制
- **规模化运作**：矩阵号、社群裂变、KOL节点，不是散点

### 三条主线

| 主线 | 典型形态 | 危害层级 | 关联体系 |
|------|----------|----------|----------|
| **伪科普→电诈** | 量子/石墨烯/负离子产品推广、伪养生号矩阵、保健品骗局 | 财产安全 | 公安反诈 |
| **伪科普→科技风评** | 核电恐慌、转基因妖魔化、5G致癌、AI威胁论、芯片自研无用论 | 国家战略 | 2035科技强国 |
| **伪科普→邪教属性** | 能量疗愈、灵修社群、反疫苗运动、伪量子意识 | 精神控制 | 国安/反邪教 |

三条线不是并列关系，往往**交叉**：一个"量子能量水"产品可能同时涉及电诈（骗钱）+ 科技风评（歪曲量子力学）+ 邪教属性（社群封闭传播、精神依赖）。

## 二、监测机制

### 2.1 四层监测架构

```
第一层：内容采集层
  ↓ 全网公开内容持续抓取
第二层：识别判定层
  ↓ AI + 科学知识图谱，判定是否伪科普及所属主线
第三层：传播分析层
  ↓ 追踪传播路径、节点、扩散速度
第四层：预警输出层
  → 实时预警 + 定期报告 + 易感人群画像
```

### 2.2 第一层：内容采集

**采集范围（按可行性分级）**

| 平台 | 数据获取方式 | 可行性 | 优先级 |
|------|-------------|--------|--------|
| 微博 | 开放API + 搜索 | ★★★★★ | P0 |
| 微信公众号 | 搜狗/新榜/清博接口 | ★★★★ | P0 |
| B站 | API + 弹幕 | ★★★★ | P0 |
| 知乎 | 搜索 + 话题追踪 | ★★★★ | P0 |
| 抖音 | 第三方数据服务 | ★★★ | P1 |
| 快手 | 第三方数据服务 | ★★★ | P1 |
| 小红书 | 第三方数据服务 | ★★ | P1 |
| 百度贴吧 | 搜索 + 爬取 | ★★★ | P1 |
| 今日头条 | 第三方数据服务 | ★★★ | P2 |
| 微信群/朋友圈 | 不可行（暗社交） | ★ | 二期 |

**采集策略**
- **关键词种子库**：维护一个持续更新的伪科普高频词库（量子能量、细胞再生、基因修复、暗物质养生……）
- **账号追踪库**：已标记的伪科普矩阵号持续追踪
- **热点事件触发**：当科技热点事件发生时（如核废水排放），自动扩大相关领域监测密度

### 2.3 第二层：识别判定

这是整个系统的**核心壁垒**。市面上的舆情系统能做情感分析、能做热度追踪，但**不能判断一条科普内容是否违反科学共识**——因为这需要领域知识。

**识别模型设计**

```
输入：一条内容（文本 / 视频转文字 / 图文OCR）
  ↓
预筛选：是否涉及科学话题？（LLM 分类）
  ↓ 是
科学主张提取：这条内容做了什么科学主张？（LLM 结构化抽取）
  例："石墨烯内裤能治前列腺炎"
  → 主张：石墨烯材料具有治疗前列腺炎的医学效果
  ↓
事实核查：该主张是否有科学依据？（知识图谱 + LLM 推理）
  → 查询：石墨烯医学应用 → 无临床证据支持该主张
  ↓
意图判定：传播动机属于哪条主线？
  → 关联商品链接 → 电诈主线
  → 涉及国家重大科技项目负面解读 → 科技风评主线
  → 来源为封闭社群/重复传播 → 邪教属性主线
  ↓
输出：
  - 伪科普概率（0-1）
  - 所属主线（可多标）
  - 科学主张 vs 科学共识的偏差描述
  - 严重等级（提示/预警/紧急）
```

**科学知识图谱**

不是从零建——对接已有权威源：
- 科普中国自有科普内容库（正样本）
- PubMed / 中国知网摘要（医学健康类）
- 国家标准库（食品安全、电磁辐射标准等）
- 科学辟谣平台历史数据（已确认的谣言，负样本）

知识图谱的作用：给 LLM 一个**事实锚点**，不靠模型自己"知道"什么是对的，而是有据可查。

### 2.4 第三层：传播路径分析

**可做的**
- **微博**：转发链完整可追溯 → 可还原传播树，识别关键传播节点（KOL/大V/营销号）
- **跨平台溯源**：同一条伪科普内容在不同平台出现 → 文本相似度匹配 → 还原跨平台传播路径
- **时间线分析**：首发平台 → 扩散平台 → 二次加工平台，追踪传播周期

**做不到但可以标注的**
- 微信群内传播（暗社交）：无法直接监测，但可以通过"微信公众号阅读量异常飙升但微博/知乎无热度"来反推群传播
- 抖音/快手的推荐算法放大效应：可以通过播放量/互动比异常来推测

**传播特征指标**
- 扩散速度（首发→N个平台的时间差）
- 节点集中度（是否由少数账号主导传播）
- 内容变异度（传播过程中是否被加工、添油加醋）
- 矩阵特征（多个账号是否同步发布高度相似内容）

### 2.5 第四层：预警与输出

**实时预警**
- 触发条件：伪科普概率 > 阈值 + 传播速度 > 阈值
- 预警等级：提示（单条内容）→ 预警（规模传播中）→ 紧急（已形成公众事件）
- 输出渠道：飞书/企微通知 → 短信 → 电话（按等级升级）

**定期报告**
- 周报：本周伪科普内容 Top 20、新增伪科普矩阵号、各主线活跃度
- 月报：传播路径分析报告、易感人群画像更新、趋势预判
- 专题报告：重大科技事件后的专项监测报告（如某核电站事件后反核传播追踪）

## 三、易感人群画像

### 方法论

不是"谁看了这条内容"——是**"什么样的人在主动传播和深度互动"**。

**数据来源**
- 互动行为：转发/评论/点赞/收藏伪科普内容的用户
- 用户画像：平台公开信息（昵称、简介、发布内容、关注列表）
- 行为模式：频次、时间段、跨平台行为

**画像维度**

| 维度 | 指标 |
|------|------|
| 人口统计 | 年龄段推断、地域分布、性别倾向 |
| 关注偏好 | 关注的账号类型（养生/宗教/阴谋论/反科学社群） |
| 传播角色 | 被动接收者 / 主动转发者 / 二次创作者 / 核心节点 |
| 易感类型 | 健康焦虑型 / 技术恐惧型 / 阴谋论倾向型 / 利益驱动型 |
| 转化风险 | 是否有购买行为（跳转电商链接）、是否加入封闭群 |

### 三条主线的典型易感画像

**电诈主线**：50+ 年龄段，关注养生/中医/家庭健康类账号，活跃时段早晨6-8点，高频转发带商品链接内容

**科技风评主线**：画像更分散——有真实的技术恐惧人群（核电周边居民），也有组织化传播（境外信息战节点），需要区分

**邪教属性主线**：封闭社群内高频互动，发布内容高度同质化，使用特定话术体系（"频率""振动""觉醒"），关注列表重合度极高

## 四、落地路径评估

### 你和我能做什么

| 模块 | 自建可行性 | 说明 |
|------|-----------|------|
| 关键词库 + 种子账号库 | ✅ 完全可以 | 人工策展 + LLM 辅助扩展 |
| 微博数据采集 | ✅ API 可用 | 需申请开发者权限 |
| 微信公众号采集 | ⚠️ 需第三方 | 新榜/清博 API，有成本 |
| 视频平台采集 | ⚠️ 需第三方 | 抖音/快手/B站，接口受限 |
| LLM 识别判定 | ✅ 核心能力 | 我来做，这是最强的环节 |
| 科学知识图谱 | ✅ 可逐步建 | 对接科普中国数据库 + 公开论文 |
| 传播路径分析 | ⚠️ 部分可做 | 微博可做，其他平台受限 |
| 易感人群画像 | ⚠️ 部分可做 | 依赖平台公开数据丰富度 |
| 实时预警 | ✅ 可以 | 飞书/企微通知 |
| 报告生成 | ✅ 核心能力 | 自动化报告 + 人工审核 |
| 7×24监测 | ⚠️ 需要基础设施 | 需要持续运行的服务器/定时任务 |

### 最小可行版本（MVP）

**你和我两个人能跑起来的版本**：

1. 聚焦微博 + 微信公众号（数据获取最可行的两个平台）
2. 维护关键词种子库，定时抓取
3. LLM 做识别判定 + 主线分类
4. 每周自动生成监测报告
5. 飞书推送预警

**不需要外包的**：识别模型、知识图谱、报告生成
**可能需要外包/采购的**：多平台数据采集接口、服务器资源

### 预算量级估算

| 项目 | 月成本 |
|------|--------|
| 第三方数据接口（新榜/清博） | ¥3,000-8,000 |
| 服务器（监测+存储） | ¥500-2,000 |
| LLM API 调用 | ¥1,000-3,000 |
| **合计** | **¥4,500-13,000/月** |
