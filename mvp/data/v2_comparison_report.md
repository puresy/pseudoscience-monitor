- 降级10次主要是**辟谣帖被规则引擎误判**后由LLM纠正
- 升级2次是规则引擎遗漏的伪科普被LLM捕获

## 3. 辟谣帖误报率

| 指标 | v1 | v2 | 变化 |
|------|----|----|------|
| 辟谣帖检出数 | -（无此功能） | 28 | 新增 |
| 辟谣帖误判为MEDIUM+ | 10 | 1 | **-9** |
| 辟谣帖误判率 | 10/28=35.7% | 1/28=3.6% | **↓90%** |

**v2辟谣帖保护机制：**
1. 规则层：辟谣关键词检测 + 账号名检测 → 高置信度辟谣帖不超过MEDIUM
2. LLM层：GLM判断is_debunking=true → 降级到LOW
3. 两层保护将辟谣帖误判率从35.7%降至3.6%

## 6. v2规则调整摘要

| 规则 | v1条件 | v2条件 | 理由 |
|------|--------|--------|------|
| R1 恐惧营销 | 恐惧词≥2 + 紧迫感≥1 | 恐惧词≥1 + 紧迫感≥1 | 单个恐惧词+紧迫感已足够可疑 |
| R2 权威虚构 | 虚假权威≥1 **且** 见证词≥1 | 虚假权威≥1即可触发(MEDIUM) | 很多伪科普只引用虚假权威，不一定有见证词 |
| R4 因果简化 | 绝对化词≥2 | 绝对化词≥1 | 配合疾病关联，1个绝对化词就够 |
| 风险阈值 | HIGH≥6.0, MEDIUM≥4.0 | HIGH≥5.0, MEDIUM≥3.0 | 配合LLM二次判断，适度放宽入口 |

## 7. 推荐阈值配置

当前v2配置已验证有效，推荐作为生产配置：

```yaml
analyzer:
  risk_thresholds:
    critical: 8.0
    high: 5.0
    medium: 3.0  # 从4.0降到3.0，让更多可疑内容进入LLM判断
    low: 0.0

llm:
  provider: zhipu
  model: glm-4-flash  # 免费模型，已验证效果
  enabled: true
```

- ✅ HIGH检出率提升 1 → 8（+7条）
