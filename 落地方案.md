> 我们能做什么、怎么做、预计效果、大概成本

## 一、我们做什么

在科学辟谣平台现有"发现—研判—创作—传播"机制基础上，建设**涉科谣言智能监测预警系统**，补齐三个缺口：

| 现有短板 | 我们补什么 |
|---------|-----------|
| 源头感知滞后，靠人工发现 | AI 7×24小时自动巡检+智能预警 |
| 只能判真假，不能判性质和意图 | 三条主线分类（电诈/科技风评/邪教属性） |
| 辟谣大水漫灌，不知道该给谁看 | 易感人群画像，支撑精准推送 |

## 二、怎么做

### 2.1 总体思路

```
全网采集 → AI研判（核心） → 传播追踪 → 预警+报告
```

不建大平台，不搞重基建。用**AI Agent + 第三方数据接口 + 轻量工程**的方式快速跑起来。

### 2.2 数据采集

| 平台 | 接入方式 | 说明 |
|------|---------|------|
| 微博 | 开放API | 转发链可追溯，传播分析最佳平台 |
| 微信公众号 | 新榜/清博API | 涉科谣言重灾区，必须覆盖 |
| B站 | API + 弹幕 | 科普/伪科普视频主阵地 |
| 知乎 | 搜索+话题追踪 | 伪科普"知识包装"高发区 |
| 抖音/快手 | 第三方数据服务 | 视频类伪科普，二期扩展 |
| 小红书 | 第三方数据服务 | 养生/美容伪科普集中，二期扩展 |

采集不是全量爬——是**种子关键词 + 重点账号追踪 + 热点事件触发**三管齐下，精准采集涉科内容。

关键词种子库基于科学辟谣平台现有6680条辟谣内容提取，初始规模2000+关键词组合，持续扩展。

### 2.3 AI智能研判（核心能力）

这是我们最强的环节，也是市面上舆情系统做不了的。

**为什么做不了？** 因为判断"一条内容是不是伪科普"不是NLP问题，是**科学事实判定问题**——需要知道"石墨烯能不能治病""5G基站的辐射量是多少""量子纠缠和人体有没有关系"。通用舆情系统没有这个知识。

**我们怎么做？**

大语言模型 + 科学知识图谱，两条腿走路：

- **知识图谱**：对接科学辟谣平台谣言库/辟谣库（6680条）、PubMed、国标库、学会专家共识——给AI一个事实锚点，不靠模型"猜"
- **大语言模型**：做科学主张提取、事实核查推理、性质判定、主线分类

**研判流程**

```
一条内容进来
  → 是否涉科？（预筛，过滤噪音）
  → 提取科学主张（"XX能治YY""XX有害"）
  → 查知识图谱，有无科学依据？
  → 判性质：普通误传 / 认知误区 / 伪科普 / 恶意造谣
  → 伪科普进一步分主线：电诈 / 科技风评 / 邪教属性
  → 定严重等级：提示 / 预警 / 紧急
```

**谣言分类标准（与现有辟谣工作对齐）**

| 类型 | 特征 | 处置 |
|------|------|------|
| 认知误区 | 非恶意，知识过时或误解 | 科普引导 |
| 冷饭热炒 | 老谣言重新传播 | 辟谣库匹配→推送已有辟谣 |
| 新发谣言 | 无先例 | 推专家研判 |
| 伪科普→电诈 | 有商品导流、诈骗特征 | 预警+对接反诈 |
| 伪科普→风评 | 针对国家科技战略 | 预警+专项监测 |
| 伪科普→邪教属性 | 封闭社群、精神控制 | 紧急+对接相关部门 |

### 2.4 传播路径分析

- **跨平台溯源**：同一谣言在不同平台的变体传播，通过语义匹配还原
- **传播时间线**：首发 → 扩散 → 二次加工 → 长尾，完整还原
- **关键节点识别**：谁在放大传播？营销号、KOL、水军矩阵
- **矩阵检测**：多账号同步发布相似内容→有组织行为
- **私域渗透推断**：公域无热度但阅读量异常→反推微信群传播

### 2.5 易感人群画像

基于伪科普内容互动者的公开行为数据，做群体画像（不涉及个人识别）：

| 主线 | 典型画像 |
|------|---------|
| 电诈 | 50+，养生/中医/家庭健康账号关注者，早间活跃，高频转发带商品链接 |
| 科技风评 | 分散——真实技术恐惧人群 vs 组织化传播节点，需区分 |
| 邪教属性 | 封闭社群高频互动，特定话术（"频率""振动""觉醒"），关注列表高度重合 |

画像用途：辟谣内容精准推送 + 反诈重点保护人群识别 + 科普选题洞察。

### 2.6 预警与报告

| 输出 | 频率 | 内容 |
|------|------|------|
| 实时预警 | 7×24 | 伪科普内容达到传播阈值时推送（飞书/企微/短信分级） |
| 周报 | 每周 | 涉科谣言Top20、新增伪科普账号、各主线活跃度 |
| 月报 | 每月 | 传播路径典型案例、易感人群画像更新、趋势预判 |
| 专题报告 | 按需 | 重大科技事件后的专项谣言追踪 |

## 四、团队与分工

| 角色 | 职责 |
|------|------|
| 产品（1人） | 需求定义、监测规则设计、报告质量把控、与科普中国对接 |
| 开发工程师（1人） | 数据采集管道、定时任务、系统搭建与运维 |
| AI Agent（1套） | 内容识别判定、知识图谱维护、传播分析、报告自动生成 |

3人配置，其中AI Agent承担传统团队中"数据分析师+AI工程师+报告撰写"的角色，大幅压缩人力成本。

专家研判依托科学辟谣平台现有专家库（2965人），不额外配置。
